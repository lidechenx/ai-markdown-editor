# AI Markdown 新手编辑器

[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](./LICENSE)

在浏览器里使用的 **Markdown 编辑器**：一边写一边看排版效果，支持粘贴截图、粘贴代码、可选的 AI 润色，以及导出 Markdown / 另存为 PDF。数据默认保存在本机浏览器，无需注册账号。

若你通过源码自行部署或参与开发，仓库地址见文末「反馈与源码」。

---

## 能做什么

- **实时预览**：左右或上下分栏查看渲染效果，可互换预览侧、拖动中间条调节宽度。  
- **写 Markdown**：用工具栏插入标题、列表等模板（见侧栏「语法速查」），也可手写语法。  
- **贴图片**：截图或图片文件粘贴/拖入后，会插入一段标准 Markdown；原文里用简短「芯片」显示，**完整图片在预览里查看**。  
- **贴代码**：直接粘贴即可；也可用侧栏插入带语言的代码块模板。  
- **（可选）AI 润色**：在侧栏配置接口地址、模型和 API Key 后，可选中文本请求润色（Key 只存在你的浏览器里）。  
- **导入 / 导出**：打开本地 `.md` 文件继续编辑；导出当前内容为 `document.md`；通过打印对话框将预览内容**另存为 PDF**。

## 如何使用（本地运行）

适合：你想在自己电脑上跑一个网页版编辑器。

```bash
git clone https://github.com/lidechenx/ai-markdown-editor.git
cd ai-markdown-editor
npm install
npm run dev
```

在浏览器中打开终端里显示的地址（一般为 `http://localhost:5173`）。

若你已把构建结果 `dist/` 部署到自己的静态网站，在浏览器访问该站点即可使用，**无需**再执行上述命令。

### 自己托管生产版本

```bash
npm install
npm run build
```

将生成的 `dist/` 目录上传到你的静态托管（如 GitHub Pages、任意 Nginx 静态目录等）。

## 界面与操作说明

1. **分栏**：顶部可切换「预览在左 / 在右」；拖动中间分隔条调整原文与预览的宽度；窗口较窄时会自动变为上下布局。  
2. **语法速查 / AI 侧栏**：顶部按钮可分别隐藏或显示；都隐藏时中间编辑区域更宽。  
3. **图片**：在编辑器内聚焦后使用 `Ctrl+V` 粘贴截图，或拖入图片文件。**请勿在他人或公共电脑上**粘贴含隐私的图片。  
4. **AI 润色**：需自行准备兼容 OpenAI 的 API 服务；在侧栏填写 Base URL、模型名和 Key。  
5. **PDF**：点击「导出 PDF（打印）」，在系统打印对话框中选择「另存为 PDF」或打印到打印机。

## 数据与隐私（使用前请读）

- 正文、界面布局、AI 配置等默认保存在浏览器的 **localStorage**。**清除本站点数据或换浏览器会丢失**，请重要内容定期用「导出 Markdown」备份。  
- 应用**不会**主动把正文上传到作者服务器；开启 AI 时，请求由你的浏览器发向你填写的接口地址。

---

## 使用上须知的限制

- **没有内置图床**：图片以数据形式写进 Markdown 时，文件体积会变大，适合草稿；长期大量配图建议以后改用图床链接（若产品后续提供相关能力，以更新说明为准）。  
- **没有账号与云同步**：换设备需自行拷贝 `.md` 文件。  
- **超大文档**：为保持流畅，特别长的文稿在原文区的「图片芯片」装饰可能会被关闭，**不影响**预览与导出内容本身。
